### 架构设计

- **`@ffmpeg/ffmpeg@0.12.15`** 已安装到 package.json（仅 18.5 KB）— Vite 本地打包 Worker chunk，无跨域问题
- **`@ffmpeg/core`**（~24 MB WASM）运行时从 jsDelivr CDN 加载，不进入 bundle
- **单例管理模式**：`ffmpegManager.ts` 全局唯一实例，`getFFmpeg()` 返回实例或 null，`resetFFmpeg()` 清空状态用于重试
- **CDN 缓存重置**：由 `src/utils/cacheReset.ts` 统一管理（ffmpeg + ECharts）

### 核心转换流程（两遍 palette 量化）

```
Step 1: fetch(videoUrl) → Uint8Array → ffmpeg.writeFile('input.mp4')
Step 2: ffmpeg.exec([-i input.mp4, -vf "scale,fps, palettegen", palette.png])
Step 3: ffmpeg.exec([-i input.mp4 -i palette.png, -lavfi "paletteuse", output.gif])
Step 4: ffmpeg.readFile('output.gif') → Blob → createObjectURL → 预览 + 下载
Step 5: 清理虚拟文件系统（deleteFile）
```

### 已修复问题

#### 1. 初始加载 UI 重构（紧凑布局）
- **问题**: ffmpeg 核心加载/错误状态使用独立 `operation-area` 卡片，视觉上不协调
- **修复**: 使用 `.core-controls` 紧凑布局紧跟 header：
  - idle → 加载按钮 + 提示文字一行
  - loading → `<el-progress>` 进度条 + 消息
  - error → `<el-alert>` 横幅 + 重试按钮（调用 `retryLoad()`，先 `resetFFmpeg()` 再重新加载）

#### 2. 自动检测 ffmpeg 缓存
- **问题**: SPA 路由切换后，如果 ffmpeg 已被其他页面或之前访问加载，仍需手动点击加载按钮
- **修复**: `onMounted` 时检查 `getFFmpeg()`，若实例已存在直接切换 `readyState = 'ready'`

#### 3. 修复"100% 后无法下载 GIF" bug
- **根因**: `new Blob([data.buffer as ArrayBuffer], ...)` — ffmpeg.wasm `readFile()` 返回的 Uint8Array 其 `.buffer` 是整个 WASM 线性内存（数 MB），不是 GIF 文件本身的字节。Blob 包含大量垃圾数据，浏览器无法渲染/下载
- **修复**: `new Blob([data.slice()], ...)` — `slice()` 创建新 `Uint8Array` 只复制有效字节，且类型正确满足 TypeScript 5.9 的 `BlobPart` 约束
- **附加**: 进度条切换逻辑改为 `v-if="converting || gifUrl"`，分离转换中/已完成视图

#### 4. TS 构建错误修复
- **`retryLoad` 未使用**: 将错误状态模板按钮从 `@click="loadCore"` 改为使用 `@click="retryLoad"`，分离 idle（首次加载）和 error（重置后重试）两个按钮状态
- **`Uint8Array<ArrayBufferLike>` → `BlobPart` 类型不兼容**: TypeScript 5.9 对 `SharedArrayBuffer` 的类型检查更严格，`Uint8Array.slice()` 返回标准 `Uint8Array<ArrayBuffer>`，满足 `BlobPart` 类型约束

#### 5. `<el-tag>` 状态切换闪现/抽动修复
- **问题**: `VideoToGif.vue` 和 `PythonRunner.vue` 中使用 `v-if`/`v-else-if`/`v-else` 链切换多个 `<el-tag>`。Element Plus 的 `<el-tag>` 内置了 zoom-in-center 过渡动画，导致：
  1. 旧标签执行 leave 动画期间仍在 DOM 中占据空间
  2. 新标签已在 DOM 中但位于旧标签右侧（flex 流顺序）
  3. 旧标签 leave 完毕完全移除后，新标签从右侧"闪现"到左侧
- **根因**: Vue 的 `v-if`/`v-else-if` 链对每个分支视为独立组件，Element Plus 的 `<Transition>` 使旧组件延迟卸载，新组件在 flex 容器中位置错位
- **修复**: 将多条 `v-if` 链替换为**单个 `<el-tag>`** 绑定 `:type` 和动态文本（computed），Vue 复用同一 DOM 节点仅更新 props，完全不触发 Element Plus 的过渡动画，新旧状态原地无缝切换
- **文件**: 
  - `src/components/VideoToGif.vue`: `readyTagConfig` computed + 单 `<el-tag>`
  - `src/components/PythonRunner.vue`: `pyTagConfig` computed + 单 `<el-tag>`

## 项目架构关键决策

### ffmpeg.wasm 加载策略
- `@ffmpeg/ffmpeg@0.12.15` 在 package.json 中，Vite 本地打包 Worker 为同域 chunk
- `@ffmpeg/core` ESM WASM（~24MB）从 CDN 加载，Worker 内 `import(CDN_URL)` 同域加载
- 单线程核心（无需 SharedArrayBuffer），无 `Cross-Origin-Embedder-Policy` 要求

### ECharts CDN 懒加载
- ECharts（~1.1MB）从 CDN 加载，不进 bundle
- `src/utils/echarts/echartsManager.ts` + `config.ts` 管理懒加载
- TreeVisualizerView chunk 从 452 KB → 15.7 KB

### 缓存重置系统
- `src/utils/cacheReset.ts` 集中管理所有 CDN 懒加载模块状态重置
- App.vue 侧边栏含"清除缓存"按钮，使用 `ElMessageBox.confirm` 二次确认
- 重置 ffmpeg + ECharts 两模块（后续新增模块在此注册）

### 共享工具 Shell 样式
- `src/style.css` 提取了统一工具容器样式（`.tool-shell`, `.shell-header`, `.operation-area`, `.action-bar`, `.panel-divider` 等）
- 所有工具共享 `.unified-shell` 垂直布局模式
- 响应式规则覆盖移动端 el-row gutter 溢出等问题
- 通用间距工具类（`.mt-4`, `.mb-4`, `.mt-2`, `.mb-2`）统一在 `style.css` 中，各组件 scoped 中不再重复定义

## 设计约定

- **无后端**，所有计算在浏览器内（WASM / CDN 懒加载）
- **无 `any` 类型**，`@ts-ignore` / `@ts-expect-error` 禁止使用（Vite 8 + vue-tsc 严格模式）
- **单线程 ffmpeg core** — 不需要 SharedArrayBuffer / COEP 头
- **Element Plus** UI 组件库，`<script setup>` 组合式 API
- **Bugfix 不改重构**，围绕最小改动
- **View 层薄**，组件层厚 — View 仅做路由包装，业务逻辑在 Component 中
- **双缓存** — 编辑器状态：内存（工具切换即时恢复）+ localStorage（页面刷新保留），每 2 分钟 + beforeunload 刷新
- **页面切换淡入淡出** — `<router-view>` 用 `<transition name="page-fade" mode="out-in">` 包裹，消除旧页面残留和新页面滑动闪烁
